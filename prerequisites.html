Onboarding Prerequisites

Everything we need from the demand partner, the publisher, and in the GAM tag before a Bidding Player tag can serve — for both instream and outstream placements.

Instream Outstream Internal · Ad Ops

Overview & ownership

A placement only serves when three sources line up. Gather all three before generating the tag:

Demand partner

The SSP/bidder (e.g. IncrementX, Limelight). Provides bidder params + must allowlist the domain and provision the right video format.

Publisher

The site owner. Provides domain, mount location, consent setup, and (instream only) a content video.

GAM

The ad server. Master video tag + key-values + VAST creative + price-bucket line items.

Most common launch blocker

Two things cause ~90% of "no ads" reports: (1) the publisher domain isn't allowlisted at the SSP, and (2) the SSP placement ID is the wrong video format (instream ID used on an outstream unit, or vice-versa). Confirm both up front.

1. From the demand partner (SSP / bidder)

  • NeedPrebid bidder code (e.g. incrementx, limelightDigital) — must exist in our bidder catalog and in the Prebid bundle at data-prebid-url.
  • NeedRequired params for that SSP (e.g. placementId, or publisherId/adUnitId/host).
  • CriticalPlacement ID provisioned for VIDEO and the correct context — an instream ID for instream, an outstream ID for outstream. Many SSPs (IncrementX included) issue format-specific IDs; a mismatch = no-bid.
  • CriticalPublisher domain allowlisted at the SSP. Without it, every auction returns no-bid.
  • OptionalFloor price / deal-ID expectations, if any.

2. From the publisher

  • NeedHTTPS pages — mixed content blocks IMA + bidder calls.
  • NeedDomain(s) to be allowlisted with each SSP.
  • NeedMount location — where the player sits. Default auto-creates a div at the script position; or they supply an existing <div id> (maps to data-div-id).
  • NeedPrivacy / consent framework — GDPR/TCF (EU; needs a CMP), CCPA/USP, or GPP. Engine defaults are tuned for non-EU; EU / US-state publishers must wire a CMP.
  • InstreamContent video URL (.mp4 / .m3u8) on their CDN — required for instream, not used for outstream.
  • OptionalAd disclosure label ("Advertisement"/"Sponsored") if their policy requires — the engine does not add one.

3. GAM tag & setup

Master video ad tag URL

  • MustEnds with correlator= (engine appends a unique correlator per impression).
  • MustContains output=vast, env=vp, gdfp_req=1, unviewed_position_start=1, and sz= with video sizes (e.g. 640x480).
  • MustCorrect iu (ad unit path).

One-time GAM configuration

  • NeedKey-values: hb_pb, hb_bidder, hb_uuid, hb_size, hb_format, hb_adid, hb_source.
  • NeedVAST wrapper creative using %%PATTERN:hb_uuid%% → Prebid Cache URL.
  • NeedPrice-bucket line items at $0.10 increments + a house fallback line item targeting hb_pb=0.00.
  • NeedLine items wrapped in one Order for easy pause/activate.
Exact string match

Prebid emits two-decimal CPMs ("2.50"). A line item targeting "2.5" will never serve.

Instream — checklist

  • SSPBidder code + params + instream video placement ID, domain allowlisted.
  • PublisherContent video URL (required), HTTPS, mount location, consent setup.
  • GAMMaster video tag (ends correlator=) + key-values + VAST creative + price-bucket line items.
  • OptionDecide if sticky / floating player is wanted (keeps the player visible on scroll).

Outstream — checklist

  • SSPBidder code + params + outstream video placement ID, domain allowlisted.
  • PublisherHTTPS, mount location, consent setup. No content video.
  • GAMMaster video tag + key-values + VAST creative + price-bucket line items.
  • NoteAutoplays muted on scroll-into-view, shows a tap-for-sound control, and collapses when there's no ad — so an empty slot showing nothing is expected, not a bug.

Pre-launch checklist (every placement)

  • Domain allowlisted at every SSP being used.
  • SSP placement ID matches the format (instream vs outstream).
  • GAM line items targeting hb_pb exist at $0.10 increments, plus a house fallback.
  • VAST wrapper creative uses %%PATTERN:hb_uuid%%.
  • Tag tested with ?debug=true — auction runs, GAM responds, ad plays.

"No ads showing" — check in this order

  1. SSP domain allowlist done?
  2. SSP placement ID matches the format (instream vs outstream)?
  3. GAM unit has line items + creatives that fill (and a house fallback)?
  4. VAST creative uses %%PATTERN:hb_uuid%%?
  5. Open the page with ?debug=true and read the console: STEP 1 → STEP 7. If it stops before STEP 7, it's a fill problem (demand/GAM), not the player. For outstream, ?debug=true also logs the IMA ad-error reason.
Rule of thumb

Reaching STEP 7 = the player works and an ad loaded. No STEP 7 = nobody filled the impression — chase demand/GAM, not the tag.
